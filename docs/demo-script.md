# ProofLoop Demo Script

> **用途**: 产品演示、客户展示、内部培训  
> **预计时长**: 15-20 分钟  
> **前置条件**: 项目已启动，测试 CSV 已准备

---

## 一、启动项目

### 1.1 安装依赖

```bash
cd insightpm-ai/apps/web
npm install
```

### 1.2 启动开发服务器

```bash
npm run dev
```

### 1.3 访问地址

- 官网: http://localhost:3000
- 控制台: http://localhost:3000/dashboard
- 新建分析: http://localhost:3000/new-analysis

---

## 二、演示流程

### Step 1: 官网首页（2 分钟）

**访问**: http://localhost:3000

**讲解要点**:
- ProofLoop 是 AI 驱动的用户反馈分析工具
- 核心价值：把用户反馈变成可信产品洞察
- 43 项硬性校验确保结论可信
- 证据链追溯支持管理层质询

**操作**:
1. 展示 Hero 区域
2. 滚动查看核心能力
3. 点击"开始分析反馈"进入登录页

---

### Step 2: 登录 Demo（1 分钟）

**访问**: http://localhost:3000/login

**讲解要点**:
- 当前为 Demo 模式
- 无需真实账号，直接进入

**操作**:
1. 输入任意邮箱（如 demo@proofloop.com）
2. 输入任意密码
3. 点击"登录"
4. 自动跳转到控制台

---

### Step 3: 控制台 Dashboard（2 分钟）

**访问**: http://localhost:3000/dashboard

**讲解要点**:
- 运行概览：已采纳数据集、硬性校验通过率、待处理报告

1. 展示统计卡片

---

### Step 4: 新建分析（3 分钟）

**访问**: http://localhost:3000/new-analysis

**讲解要点**:
- 支持 CSV/JSON 格式上传
- 可配置场景类型
- 自动触发 AI 分析流程

**操作**:
1. 输入案例名称（如 "demo-test"）
2. 选择场景（如 "企业 SaaS 续费"）

5. 点击"开始分析"
6. 等待分析完成（约 1-2 分钟）
7. 展示分析结果

**测试 CSV 示例**:
```csv

"登录页面加载太慢，每次要等5秒",性能问题,负面,高,App Store
"新功能很棒，但文档不够详细",文档,中性,中,客服反馈
"支付流程经常报错，影响转化",支付问题,负面,高,用户访谈
```

## 三、常见问题 Q&A

### Q1: 数据安全如何保障？

**A**: 所有数据处理在本地完成，不上传至外部服务器。SaaS 版本将提供端到端加密与 SOC 2 合规。

### Q2: 支持哪些数据格式？

**A**: 目前支持 CSV 和 JSON 格式。未来将支持更多格式和 API 直接接入。

### Q3: 分析准确率如何？

**A**: 硬性校验覆盖 43 项规则，确保数据完整性。语义校验通过 AI 交叉验证，当前平均语义评分达 88%。

### Q4: 是否支持团队协作？

**A**: 当前为本地 Demo 模式。SaaS 版本将支持多成员协作、权限管理与实时通知。

### Q5: 如何自定义分析场景？

**A**: 在"系统设置 → 自定义场景"中可以创建和管理自定义场景。

## 四、演示注意事项

### 4.1 演示前检查

- [ ] 项目已启动（`npm run dev`）
- [ ] 测试 CSV 文件已准备
- [ ] 浏览器已打开（推荐 Chrome）
- [ ] 网络连接正常（AI 模型调用需要）

### 4.2 演示技巧

1. **讲故事**: 从真实用户反馈场景切入
2. **对比**: 展示传统方法 vs ProofLoop 的效率差异
3. **互动**: 让观众提供反馈数据进行现场分析
4. **留白**: 每个功能点讲解后留 10 秒让观众消化

### 4.3 风险预案

| 风险 | 预案 |
|------|------|
| AI 分析超时 | 准备已完成的分析结果作为备用 |
| 页面加载慢 | 提前预热页面，使用已完成的 run |
| 网络中断 | 准备离线截图或 PDF |
